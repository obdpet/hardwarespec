# Carpybara — Trade Study #2: 디스플레이 선정

> **결정 사항**: Carpybara의 양산 디스플레이를 어떤 크기/드라이버 IC/패널 타입으로 확정할 것인가?  
> **작성일**: 2026-04-14  
> **제약 조건**: 송풍구 클립 케이스(≤2.8") / BOM 디스플레이 단가 ~$3~5 (10K) / 70°C 동작 온도 + SW 보호

---

## 0. Trade Study란?

하드웨어 엔지니어가 부품이나 설계 방향을 결정할 때 사용하는 **표준 의사결정 방법론**입니다.

핵심 원리:
1. **감으로 고르지 않는다** — 평가 기준을 먼저 정하고, 점수로 비교한다
2. **가중치가 결론을 만든다** — 같은 부품이라도 프로젝트 상황에 따라 정답이 달라진다
3. **문서로 남긴다** — "왜 이걸 골랐는가"를 나중에 설명할 수 있어야 한다

> Trade Study는 "정답을 찾는 도구"가 아니라 **"오답을 걸러내는 도구"**입니다.
> 최종 판단은 여전히 사람이 하지만, 근거가 투명하게 남습니다.

---

## 1. 문제 정의 (Problem Statement)

Carpybara는 OBD2 데이터를 읽어 캐릭터 애니메이션을 출력하는 차량용 가젯이다.  
프로토타입은 Waveshare 2.0" ST7789로 동작 확인 완료.  
Frozen AVL은 2.4" ILI9341을 양산 후보로 지정했으나, 공급업체가 미확정(PENDING) 상태다.  
LLM 비교 리뷰에서 디스플레이 공급망이 **단일 실패점(SPOF)**으로 경고받았다.

**결정해야 할 것**: 양산 디스플레이를 어떤 크기/드라이버 IC/패널 타입으로 확정하고, 2nd 소스 전략을 어떻게 구성할 것인가?

---

## 2. 필수 요구사항 (Pass/Fail 필터)

이 조건을 **하나라도 만족하지 못하면 즉시 탈락**합니다.  
점수를 매기기 전에 후보를 걸러내는 1차 필터입니다.

| # | 필수 요구사항 | 이유 |
|---|-------------|------|
| R1 | **SPI 인터페이스, 실동작 클럭 ≥ 40MHz** | ESP32 실용 SPI는 40MHz. 이보다 낮으면 목표 FPS 미달 |
| R2 | **해상도 240×320 이상** | 캐릭터 애니메이션 캔버스와 OBD 데이터 오버레이가 이 해상도 기준으로 설계됨. 변경 시 에셋 전면 재작업 |
| R3 | **TFT_eSPI 라이브러리 공식/커뮤니티 드라이버 존재** | 펌웨어 스택이 TFT_eSPI 기반. 커스텀 드라이버가 필요한 디스플레이는 짧은 데드라인에서 리스크 |
| R4 | **동작 온도 min ≤ −20°C, max ≥ +70°C** | 송풍구 클립 설치 + SW 백라이트 차단으로 70°C 이내 제어. 이보다 낮으면 보호 로직으로도 커버 불가 |
| R5 | **대각선 크기 ≤ 2.8"** | 송풍구 클립 케이스의 물리적 크기 한계 |
| R6 | **FPC 커넥터 핀아웃 문서화** | 캐리어 보드 PCB에 FPC 풋프린트를 설계하려면 핀아웃이 반드시 공개되어야 함 |

### 필수 요구사항 필터 결과

| 후보 | R1 SPI 40MHz | R2 240×320 | R3 TFT_eSPI | R4 −20/+70°C | R5 ≤2.8" | R6 FPC 핀아웃 | 결과 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **A: 2.0" ST7789** | O (62.5MHz) | O | O (내장) | O | O | O | **PASS** |
| **B: 2.4" ILI9341** | O (40MHz+) | O | O (최고 성숙도) | O | O | O (40-pin 표준) | **PASS** |
| **C: 2.4" ST7789VI** | O (62.5MHz) | O | O (ST7789 계열) | O | O | O (Newhaven 문서) | **PASS** |
| **D: 2.8" ILI9341** | O | O | O | O | O (경계) | O | **PASS** |

> Option D의 2.8"는 케이스 크기 한계(≤2.8")의 경계선에 위치합니다. 공차 여유가 없으므로 조건부 통과.

**4개 후보 모두 Pass. 평가 매트릭스로 진행합니다.**

## 3. 후보 디스플레이 (Candidates)

---

### Option A: 2.0" ST7789 (현재 프로토타입)

```
[현재 상태]           [양산]
Waveshare 2.0"    →   2.0" ST7789 모듈 (ODM 소싱)
ST7789, IPS           (동일 드라이버, 동일 코드)
SPI 27MHz (40MHz 가능)
```

- **드라이버 IC**: ST7789 (Sitronix)
- **패널 타입**: IPS (170° 시야각)
- **핵심 공급처**: Waveshare (~$8~12), Adafruit ($14.95), BuyDisplay (문의)
- **장점**: 프로토타입에서 이미 검증됨. IPS 표준. 슬립 전류 5µA (초저전력)
- **단점**: 2.0" 크기 — 캐릭터 스테이지가 좁음. 2nd 소스 제한적. 2.4" 대비 commodity 시장 작음

---

### Option B: 2.4" ILI9341 (현재 AVL 1st 소스 후보)

```
[현재 상태]           [양산]
Waveshare 2.0"    →   TSLCD TST024QVHI-17 (IPS, ILI9341)
ST7789 프로토타입      (드라이버 변경: ST7789→ILI9341)
                       2nd 소스: LCDWiki MSP2402 IPS variant
```

- **드라이버 IC**: ILI9341 (Ilitek)
- **패널 타입**: TN이 기본, **IPS variant 지정 필수**
- **핵심 공급처**: TSLCD TST024QVHI-17 (IPS, 견적 필요), LCDWiki MSP2402 (~$5~8), DisplayModule DM-TFT24-401 ($19.90 비상용)
- **장점**: 가장 큰 commodity 시장. 10K 단가 $3~5 달성 용이. TFT_eSPI 최고 성숙도. 40-pin FPC 사실상 표준
- **단점**: ST7789→ILI9341 드라이버 IC 변경 필요. **TN 패널이 기본값** — IPS 명시 조달 실패 시 시야각 문제

---

### Option C: 2.4" ST7789VI (Newhaven Display)

```
[현재 상태]           [양산]
Waveshare 2.0"    →   Newhaven NHD-2.4-240320AF-CTXP-T
ST7789 프로토타입      (동일 드라이버 계열: ST7789→ST7789VI)
```

- **드라이버 IC**: ST7789VI (Sitronix, ST7789 계열)
- **패널 타입**: **IPS 확정** (데이터시트 명기)
- **핵심 공급처**: Newhaven NHD-2.4-240320AF-CTXP-T ($14.98, Mouser/Digi-Key 재고)
- **장점**: 최신 드라이버 IC + 2.4" 크기. IPS 보장. Mouser/Digi-Key 정규 유통. ST7789 코드 최대 연속성
- **단점**: 단가 높음 ($14.98 단품, 10K 추정 $8~12). Newhaven 외 2nd 소스 제한적. FPC 핀아웃이 ILI9341 표준과 다를 수 있음

---

### Option D: 2.8" ILI9341 (최대 화면)

```
[현재 상태]           [양산]
Waveshare 2.0"    →   2.8" ILI9341 모듈 (산업용 variant)
ST7789 프로토타입      (드라이버 변경 + 인클로저 재설계)
```

- **드라이버 IC**: ILI9341
- **패널 타입**: TN 기본 (산업용 IPS/MVA variant 존재)
- **핵심 공급처**: DisplayModule 산업용 (~$20~30), LCDWiki MSP2807 (~$7~10)
- **장점**: 최대 화면 — 캐릭터 스테이지와 OBD 데이터 여유 공간 극대화. 산업용 variant는 −30°C~+85°C 가능
- **단점**: 인클로저 크기 한계 경계선. 2.4" FPC와 호환 불가. 인클로저 CAD 재설계 필요

## 4. 평가 기준 및 가중치 (Evaluation Criteria)

> **가중치를 정하는 원칙**: 프로젝트의 가장 큰 제약이 가장 높은 가중치를 받는다.
> Carpybara 디스플레이 선정의 가장 큰 제약은 "$50 소매가에서 디스플레이 BOM $3~5"과 "공급망 SPOF 해소"이다.

| # | 평가 기준 | 가중치 (1~5) | 가중치 설정 이유 |
|---|----------|:---:|------|
| C1 | **디스플레이 크기/시인성** | **4** | 디스플레이가 곧 제품. 운전자가 팔길이에서 캐릭터를 알아볼 수 있어야 함 |
| C2 | **시야각 (IPS vs TN)** | **4** | 송풍구 설치 시 운전자 시선은 off-axis. TN은 40°+ 에서 색 반전 — 제품 경험 치명적 |
| C3 | **2nd 소스 공급망** | **4** | LLM 리뷰에서 SPOF 경고. 단일 공급처 의존 시 4주 품절 = 양산 중단 |
| C4 | **10K BOM 단가** | **4** | $50 소매가 / BOM ~$17~22 / 디스플레이 $3~5 타겟. 초과 시 마진 붕괴 |
| C5 | **TFT_eSPI 펌웨어 성숙도** | **3** | HW 경험 0 팀. 예제·튜토리얼·커뮤니티 답변이 풍부해야 막히지 않음 |
| C6 | **FPC 커넥터 표준화** | **3** | 캐리어 보드 PCB에 단일 FPC 풋프린트 필요. 1st/2nd 소스 교체 시 PCB 리스핀 불가 |
| C7 | **프로토→양산 코드 연속성** | **3** | 프로토타입이 ST7789. 드라이버 IC 변경은 init 시퀀스/컬러 포맷/회전 레지스터 재검증 필요 |
| C8 | **부분 업데이트 렌더링 성능** | **2** | 40MHz SPI에서 풀스크린 리프레시는 어느 드라이버든 ~12~15fps로 동일. 부분 업데이트(80×80) 기준 차이도 미미 |
| C9 | **인클로저 설계 영향** | **1** | 2.4" vs 2.8"는 케이스 CAD에 영향. 하지만 디스플레이 확정 후 케이스를 맞추면 되므로 낮은 가중치 |
| | **가중치 합계** | **28** | **만점 = 28 × 5 = 140** |

### C1. 디스플레이 크기/시인성 (가중치: 4)

| 후보 | 점수 | 근거 |
|------|:---:|------|

### C2. 시야각 — IPS vs TN (가중치: 4)

| 후보 | 점수 | 근거 |
|------|:---:|------|
| **A: 2.0" ST7789** | **5** | IPS 표준(Waveshare 모듈). 170° 시야각. 프로토타입에서 off-axis 시인성 직접 확인됨 |
| **B: 2.4" ILI9341** | **3** | **TN 패널이 commodity 기본값.** TSLCD TST024QVHI-17은 IPS로 명기되어 있으나, ODM 조달 시 TN이 섞일 리스크. IPS 조달을 QC로 강제해야 함. 미확인 리스크 반영하여 3점 |
| **C: 2.4" ST7789VI** | **5** | **IPS가 데이터시트에 명시.** Newhaven NHD-2.4-240320AF-CTXP-T는 공식 스펙시트에서 IPS 확정. 조달 리스크 없음 |
| **D: 2.8" ILI9341** | **3** | 2.8" commodity 모듈은 거의 전부 TN. 산업용 IPS variant는 존재하지만 단가 급등($20+). 안정적 IPS 조달 어려움 |

> **B의 핵심 리스크**: ILI9341 2.4"를 "IPS로 달라"고 주문해도 공급업체가 TN을 보낼 수 있음. **입고 시 QC 기준(60° off-axis 색 반전 여부)** 필수.

## 7. 민감도 분석 (Sensitivity Analysis)

> **민감도 분석이란**: "내가 가중치를 잘못 정했으면 어떻게 되는가?"를 확인하는 과정.
> 결론이 가중치 하나 바꿨을 때 뒤집힌다면, 그 결론은 신뢰하기 어렵습니다.

### 시나리오 1: "코드 연속성이 더 중요하다면?" (C7 가중치 3→6)

프로토타입 코드를 최대한 그대로 쓰는 것이 최우선인 경우.

| 후보 | 기존 총점 | 추가분 | 변경 후 총점 |
|------|:---:|:---:|:---:|
| A: 2.0" ST7789 | 95 | +5×3 = +15 | **110** |
| B: 2.4" ILI9341 | 121 | +3×3 = +9 | **130** |
| C: 2.4" ST7789VI | 101 | +4×3 = +12 | **113** |
| D: 2.8" ILI9341 | 96 | +2×3 = +6 | **102** |

→ **순위 변동 없음. B 여전히 1위 (130).** 코드 연속성을 2배로 높여도 B의 공급망+비용 우위를 뒤집지 못함.

### 시나리오 2: "공급망이 모든 것에 우선한다면?" (C3 가중치 4→8)

LLM 리뷰의 SPOF 경고를 극도로 심각하게 받아들이는 경우.

| 후보 | 기존 총점 | 추가분 | 변경 후 총점 |
|------|:---:|:---:|:---:|
| A: 2.0" ST7789 | 95 | +2×4 = +8 | **103** |
| B: 2.4" ILI9341 | 121 | +5×4 = +20 | **141** |
| C: 2.4" ST7789VI | 101 | +3×4 = +12 | **113** |
| D: 2.8" ILI9341 | 96 | +4×4 = +16 | **112** |

→ **B의 리드가 오히려 확대 (141). 공급망 SPOF 해소가 B의 핵심 강점.**

### 시나리오 3: "BOM 비용이 덜 중요하다면?" (C4 가중치 4→1)

소매가를 $60~65로 올리거나 마진을 줄여서라도 좋은 부품을 쓰겠다는 경우.

| 후보 | 기존 총점 | 차감분 | 변경 후 총점 |
|------|:---:|:---:|:---:|
| A: 2.0" ST7789 | 95 | −3×3 = −9 | **86** |
| B: 2.4" ILI9341 | 121 | −5×3 = −15 | **106** |
| C: 2.4" ST7789VI | 101 | −2×3 = −6 | **95** |
| D: 2.8" ILI9341 | 96 | −3×3 = −9 | **87** |

→ **B 여전히 1위 (106). C가 95로 상승하며 B와 격차 축소.** 비용 제약을 완화하면 Option C(Newhaven ST7789VI)가 "프리미엄 IPS 확정 + 코드 연속성" 경로로 매력도 상승.

### 시나리오 4: "시야각이 제품 차별화의 핵심이라면?" (C2 가중치 4→7)

"운전석에서 어떻게 보이느냐가 제품의 전부"인 경우.

| 후보 | 기존 총점 | 추가분 | 변경 후 총점 |
|------|:---:|:---:|:---:|
| A: 2.0" ST7789 | 95 | +5×3 = +15 | **110** |
| B: 2.4" ILI9341 | 121 | +3×3 = +9 | **130** |
| C: 2.4" ST7789VI | 101 | +5×3 = +15 | **116** |
| D: 2.8" ILI9341 | 96 | +3×3 = +9 | **105** |

→ **B 여전히 1위 (130). C가 116으로 격차 축소.** 이 시나리오가 시사하는 바: **B를 선택하더라도 IPS 패널을 반드시 확보해야 한다.** B의 C2 점수가 3→5(IPS 확정 조달)로 올라가면 총점이 121→129로 상승하여 어떤 시나리오에서든 압도적 1위.

### 민감도 분석 결론

**어떤 시나리오에서도 Option B가 1위를 유지합니다.** 순위가 뒤집힐 가능성은 없습니다.

단, 모든 시나리오에서 공통된 메시지가 있습니다: **B의 유일한 약점은 IPS 조달 불확실성.** IPS를 확정 조달할 수 있다면 B는 단순히 "최선"이 아니라 "압도적 최선"입니다.

## 8. 분석 및 논의

### B가 이기는 이유

Option B(2.4" ILI9341 IPS)가 1위인 핵심 이유는 **공급망 + 비용 + 펌웨어 성숙도의 삼중 우위** 때문입니다.

- **공급망**: 2.4" ILI9341은 세계에서 가장 많이 생산되는 소형 TFT 세그먼트. TSLCD, LCDWiki, DisplayModule, Alibaba ODM 등 다수 공급처가 동일한 40-pin FPC 표준을 공유. 2nd 소스 교체 시 PCB 변경 불필요
- **비용**: 경쟁이 치열한 세그먼트이므로 $3.50 (10K) 달성이 현실적. 이미 HW Spec에 이 단가가 기재되어 있음
- **성숙도**: TFT_eSPI의 "첫 번째 시민". Arduino + ESP32 + ILI9341 조합은 커뮤니티에서 가장 많이 사용되며, ELMduino OBD 대시보드 예제도 대부분 ILI9341 기반

이 Trade Study는 기존 AVL의 2.4" ILI9341 선정을 **확인(confirm)**합니다. 뒤집는 것이 아닙니다.

### B의 치명 약점: TN 기본값

Option B의 유일하지만 중요한 약점: **ILI9341 2.4" commodity 시장에서 TN 패널이 기본값입니다.**

IPS로 달라고 주문해도 공급업체가 TN을 보내는 사례가 있습니다. 이를 방지하려면:
1. RFQ(견적 요청서)에 "Panel type: IPS — TN not acceptable" 명기
2. 입고 QC: 60° off-axis에서 색 반전 여부 육안 확인 (IPS는 반전 없음, TN은 반전)
3. 1st 소스를 TSLCD TST024QVHI-17 (데이터시트에 IPS 명기)로 확정

### C가 합리적인 경우

Option C(2.4" ST7789VI, Newhaven)는 다음 상황에서 올바른 선택입니다:

1. **ILI9341 IPS 조달이 실패할 때** — TSLCD가 IPS를 공급하지 못하거나, commodity 채널에서 IPS를 안정적으로 확보할 수 없을 때
2. **BOM 비용 제약이 완화될 때** — 소매가를 $60~65로 올리거나, 디스플레이에 $8~12 투자가 허용될 때
3. **코드 연속성을 극대화하고 싶을 때** — ST7789 계열 유지로 드라이버 변경 리스크 제거

Option C는 "시스템 레벨 2nd 소스"로서의 역할이 있습니다: FPC 풋프린트가 다르므로 drop-in 교체는 불가하지만, B의 공급망이 무너졌을 때 PCB 리스핀 1회로 전환 가능한 대안입니다.

### A가 양산에 부적합한 이유

Option A(2.0" ST7789)는 프로토타입 검증 가치는 있으나, 양산 선택으로는 구조적 약점이 있습니다:

- 2.0" 크기의 commodity 시장이 2.4"보다 작음 → 2nd 소스 확보 어려움
- Waveshare 사실상 단일 공급처 → 정확히 LLM 리뷰가 경고한 SPOF 패턴
- FPC 풋프린트가 2.4"와 다름 → 프로토→양산 시 캐리어 보드 PCB 전면 리스핀

**프로토타입(EVT/DVT)에서는 Option A를 그대로 사용합니다.** 코드 검증, FPS 벤치마크, 애니메이션 테스트는 모두 2.0"에서 수행. 양산 PCB 설계 시점에 2.4"로 전환합니다.

### D가 미래 SKU인 이유

Option D(2.8" ILI9341)는 디스플레이 면적이 최대이므로 "Carpybara Pro" 같은 미래 SKU에 적합합니다. 하지만 v1에서 선택하기에는:

- 인클로저 크기 한계(≤2.8") 경계선 — 공차 여유 없음
- 2.4" FPC와 호환 불가 → PCB 별도 설계
- B와 동일한 ILI9341 드라이버이므로, 나중에 2.4"→2.8"로 확장할 때 전환 비용이 매우 낮음

## 9. 추가 고려사항

### FPC 커넥터 표준화 전략

캐리어 보드 PCB 설계 시 **단일 FPC 풋프린트**를 채택해야 합니다:

- **기준 풋프린트**: 2.4" 40-pin FPC, 0.5mm pitch (ILI9341 표준 핀아웃)
- **1st 소스**: TSLCD TST024QVHI-17 — 이 모듈의 핀아웃이 풋프린트를 정의
- **2nd 소스**: LCDWiki MSP2402 IPS variant — 동일 40-pin 핀아웃 확인 필요
- **비상 조달**: DisplayModule DM-TFT24-401 — DVT/PVT 소량 용도 ($19.90)

> **PCB 테이프아웃 전 필수 확인**: 2nd 소스 FPC 커넥터가 1st 소스와 물리적으로 동일한 풋프린트인지 **실물 모듈 2~3개**를 구매하여 직접 확인.

### IPS vs TN 조달 강제 방법

1. **RFQ 템플릿**에 "Panel type: IPS / MVA — TN not acceptable" 항목 추가
2. **입고 QC 기준**: 60° 수평각에서 화면 켜고 색 반전 확인 (IPS = 무반전, TN = 반전/워시아웃)
3. **AVL에 TN 불가 명기**: "TN 패널 납품 시 전량 반품" 조항

### 동작 온도 한계 + SW 백라이트 차단

4개 후보 모두 소비자 모듈 기준 **+70°C 동작 온도 한계**. 85°C 이상 소비자 모듈은 시장에 없음.

**대응 전략**:
- SW 백라이트 차단 임계치: **65°C** (70°C 한계에서 5°C 여유)
- 온도 감지: TEMT6000 회로 또는 NTC 서미스터 (디스플레이 FPC 근처 배치)
- 제품 리스팅에 명시: "동작 온도: −20°C ~ +65°C 연속 / +70°C 피크 (자동 열보호)"
- 송풍구 클립 설치로 직사광선 노출 최소화

### ST7789→ILI9341 드라이버 전환 가이드 (Option B 채택 시)

TFT_eSPI에서 드라이버 변경은 `User_Setup.h` 수정으로 처리:

1. `#define ST7789_DRIVER` → `#define ILI9341_DRIVER`로 변경
2. SPI 핀 매핑 재확인 (MOSI, SCLK, CS, DC, RST)
3. 컬러 바이트 오더 확인: ST7789는 RGB, ILI9341은 BGR이 기본 → `TFT_RGB_ORDER` 설정 확인
4. 회전 레지스터 동작 차이 테스트 (0/1/2/3 각 값에서 화면 방향 확인)
5. SPI 클럭: 27MHz → 40MHz로 상향 가능 (ILI9341은 40MHz 안정 구간)

**예상 소요 시간**: 0.5일

1. 총점 1위 (121/140, 86.4%) — 2위와 20점 차
2. 4개 민감도 시나리오 전부에서 1위 유지 — 결론이 견고함
3. 2.4" ILI9341은 세계 최대 commodity TFT 세그먼트 — 공급망 SPOF 해소
4. 10K 단가 $3~5 달성 가능 — BOM 타겟 충족
5. TFT_eSPI 최고 성숙도 — HW 초보 팀의 리스크 최소화
6. 40-pin FPC 사실상 표준 — 2nd 소스 교체 시 PCB 리스핀 불필요

**필수 조건**:
- **IPS 패널 반드시 지정**. TN 기본값 조달 불가. QC 기준 수립 필수
- **1st 소스**: TSLCD TST024QVHI-17 (IPS 명기, 견적/MOQ 확인 필요)
- **2nd 소스**: LCDWiki MSP2402 IPS variant (FPC 핀 호환 확인 필요)
- **비상 조달**: DisplayModule DM-TFT24-401 ($19.90, DVT/PVT 소량 전용)

### 시스템 레벨 대안: Option C — 2.4" ST7789VI (Newhaven)

**다음 경우에만 Option C로 전환**:
- B의 IPS 조달이 실패했을 때 (TSLCD가 IPS를 안정 공급하지 못함)
- 소매가를 $60+로 올릴 수 있을 때

전환 시 PCB 리스핀 1회 필요 (FPC 풋프린트 변경).

### 프로토타입: Option B와 동일 (2.4" ILI9341)

**프로토타입도 2.4" ILI9341로 진행합니다.**  
해상도(240×320)는 동일하지만, 물리적 크기 차이(6.7mm vs 8.0mm 캐릭터)가 디자인 판단에 영향을 줌.  
2.0"에서 검증한 디자인이 2.4"에서 다르게 느껴질 수 있으므로, 양산과 동일한 디스플레이로 처음부터 개발.

- Waveshare 2.4" ILI9341 모듈 구매 → 프로토타입 즉시 전환
- TFT_eSPI config 변경: ST7789→ILI9341 (0.5일)
- 기존 2.0" ST7789는 개발 스페어/디버그용으로 보관

### 비권고: Option D

미래 "Carpybara Pro" SKU에 적합. v1에서는 인클로저 공차 없음 + 별도 PCB 설계 필요로 비권고.
